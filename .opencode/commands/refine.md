# 需求结构化

只调用 `testcase-refiner` Skill，将零散的测试需求转化为标准的结构化测试步骤文档。

## 使用方式

```
/refine <你的测试需求描述>
```

## 适用场景

- 只想整理测试思路，不需要立即生成代码
- 需要评审测试场景覆盖是否完整
- 手工用例文档化

## 输出

标准格式的测试步骤文档，包含：
- 基本信息（功能模块、测试端、前置条件）
- 测试用例列表（步骤、输入数据、期望结果）

这份输出可以作为 `testcase-planner` 的输入。
